# 校园AI助手 升级重构规划报告

## 一、 项目背景与升级目标
本项目旨在对现有的“校园AI助手”进行全面升级重构。目标是将传统的“被动问答知识库”进化为“全场景、主动式的高效AI Agent”，将学生日常高频使用的教务、生活、活动等系统深度整合，提供一站式、个性化、极速的校园智慧服务。

## 二、 核心功能模块规划

### 1. 基础知识与数据底座
* **公共信息库**：整合并持续更新学校现有静态/半静态资源，包含最新学校政策、学生手册及各专业详细信息。
* **个人教务数据**：通过安全授权对接教务系统，实现个人课表、成绩、学分等私有数据的精准查询与调用。

### 2. 动态信息自动化引擎
* **定向爬虫部署**：开发并部署自动化爬虫，定期监控并抓取学校各大官网及部门网站的最新通知与政策。一直新增怎么办？如果某个知识太久被检索过，可以从知识库中删除。
* **流程替代**：彻底取代以往需要人工输入关键信息和手动维护知识库的繁琐流程，确保AI信息源的实时性与全面性。
  网站：教务处，二课，志愿者活动qq群，

### 3. 全场景服务聚合与快捷交互（待定）
* **校园系统全接入**：打通校园内各业务系统API，集成图书馆（座位预约）、第二课堂（学分/活动报名）、志愿服务平台及上课系统（请假、签到打卡）等。
* **统一 Chat 入口**：将散落于多个网页和App的业务流程，统一收口至单一对话框。
* **极速语音交互**：引入流畅的快捷语音指令，支持自然语言一键操作复杂系统（例如：“帮我请明天上午的假”）。

### 4. 个性化配置与主动式服务架构
* **专属账号体系**：以学生学号作为唯一UID，构建独立的个人数字档案。
* **自定义 AI 风格**：初始状态下AI会知道你的学院专业年级等教务系统上的静态数据。同时可以引入类似 OpenClaw 的自定义功能，用户可根据自身需求设定助手的回复风格、长度与侧重点，在每日的对话中慢慢了解学生的习性，对于学生越来越模糊的请求给出越来越正确的回复。
* **轻量级主动推送机制**：支持个性化事件订阅（如特定讲座开放报名提醒）。
  * **架构优势**：监控与资源匹配的“重计算”完全由云端服务器后台执行，不占用移动端算力与流量。
  * **触达方式**：条件触发后，后端通过自研的**钉钉传话功能**，将包含时间、地点、链接的轻量级文本推送至用户端，实现零传输负担的主动提醒。

### 5. 性能保障与极速响应
* **端到端传输优化**：手机端仅发送极简文本指令，历史上下文对话状态由云端（如 Redis）专门维护，极大降低请求体大小。
* **底层并发与检索优化**：针对 RAG（检索增强生成）流程、爬虫数据库检索以及多校园系统并发 API 请求进行深度性能调优，保障高频使用的极致流畅体验。
